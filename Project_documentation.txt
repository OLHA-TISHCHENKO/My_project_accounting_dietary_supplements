Описание WEB-приложения "ВитаминCome"


Приложение позволяет зарегистрированному пользователю вводить информацию о пищевых добавках, создавать списки пищевых добавок и отмечать в календаре дату приёма соответствующей пищевой добавки, чтобы отслеживать период приема препарата и коррелировать его с состоянием своего здоровья.


Первая страница приложения дает возможность:
* незарегистрированному пользователю перейти на страницу “Форма регистрации”. (После нажатия кнопки “Зарегистрироваться” происходит переход на страницу “Форма регистрации”)
* зарегистрированному пользователю перейти на страницу введения логина и пароля (После нажатия кнопки “Войти” происходит переход на форму введения логина и пароля).


Форма регистрации дает возможность:
*  незарегистрированному пользователю зарегистрироваться в приложении путем введения необходимой информации:
1. Имя 
2. Фамилия
3. Логин   (д.б. уникальным для базы данных)
4. Пароль (от 3 до 50 символов)
 (После нажатия кнопки “Зарегистрироваться” происходит переход на страницу “Мой Витамин”);
* вернуться на первую страницу приложения.


Форма введения логина и пароля дает возможность:
-  зарегистрированному пользователю войти в приложение (После нажатия кнопки “Войти” происходит переход на страницу “Мой Витамин”);
- вернуться на первую страницу приложения.


Страница “Мой витамин” дает возможность:
* видеть перечень принимаемых препаратов;
* Перейти на страницу “Календарь” (После нажатия кнопки “Посмотреть календарь” происходит переход на страницу “Календарь”); 
* перейти на страницу “Добавить Витамин” (После нажатия кнопки “Добавить” происходит переход на страницу “Добавить Витамин”);
* удалить пищевую добавку из списка (После нажатия на кнопку “Корзина” открывается pop-up о подтверждение удаления).


 Страница “Добавить витамин” даёт возможность:
* внести информацию о препарате (все поля обязательны к заполнению):
1. Наименование препарата;
2. Производитель;
3. Основное действующее вещество;
4. Количество действующего вещества;
5. Цвет отметок в календаре (После нажатия кнопки с изображением стрелки, в поле “Цвет отметок в календаре” открывается pop-up с вариантами цвета).
* сохранить внесенные изменения нажатием кнопки “Добавить”;
* вернуться на страницу “Мой витамин”. 


 
Страница “Календарь” даёт возможность:
* добавить отметку в календаре кликнув на необходимой дате;
* убрать отметку в календаре повторно кликнув на уже отмеченной дате;
* сохранить внесенные изменения;
* просматривать календарь с отметками о приёме препарата;
* вернуться на страницу “Мой витамин”. 
 






User story 


1 Как незарегистрированный пользователь я хочу зарегистрироваться в приложении, чтобы бы пользоваться возможностями приложения.
2  Как зарегистрированный пользователь я хочу войти в приложение чтобы пользоваться его возможностями.
3 Как зарегистрированный пользователь я хочу иметь возможность вносить необходимые мне препараты в список пищевых добавок.
4 Как зарегистрированный пользователь я хочу иметь возможность удалять препараты из списка пищевых добавок.
5 Как зарегистрированный  пользователь я хочу иметь возможность отмечать в календаре дату приёма пищевой добавки. 
6 Как зарегистрированный  пользователь я хочу видеть в календаре даты приема пищевых добавок, чтобы отслеживать период приёма препарата. 




Критерии приемки


User story 1 Как незарегистрированный пользователь я хочу зарегистрироваться в приложении, чтобы бы пользоваться возможностями приложения. 
1. На первой странице приложения есть кнопка “Зарегистрироваться”.
2. После нажатия на кнопку “Зарегистрироваться” открывается “Форма регистрации”.
3. “Форма регистрации” содержит:
* Заголовок “Форма регистрации” 
*  поле “Имя”
*  поле “Фамилия”
*  поле “Логин” 
*  поле “Пароль”
* текст “Для регистрации заполните все поля”
* кнопку “Зарегистрироваться“
* кнопку “Вернуться назад”.
4. После нажатия кнопки “Зарегистрироваться” происходит проверка наличия пользователя с таким логином в базе данных:
* Если пользователь с таким логином уже существует
на форме под полем “Логин” появляется надпись “Данный логин уже существует. Введите другой логин”, шрифт значения в поле логин становится красным.
*  Если пользователя с таким логином нет в базе данных:
новый пользователь регистрируется в приложении. 


User story  2  Как зарегистрированный пользователь я хочу войти в приложение чтобы пользоваться его возможностями.
1. На первой странице приложения есть кнопка “Войти”.
2. После нажатия кнопки “Войти” открывается форма авторизации.
3. Форма авторизации содержит:
*  поле “Логин” 
*  поле “Пароль”
* кнопку “Войти”
* кнопку “Вернуться назад”
4. После введения неверного логина или пароля над кнопкой “Войти” появляется надпись “Неверный логин или пароль. Попробуйте ещё раз”
5.  После введения верных логина, пароля и нажатия кнопки “Войти” открывается страница “Мой витамин”. 




User story 3 Как зарегистрированный пользователь я хочу иметь возможность вносить необходимые мне препараты в список пищевых добавок.
1. После нажатия кнопки “Добавить” на странице “Мой витамин” открывается страница “Добавить Витамин”.
2. Страница “Добавить Витамин” содержит:
* поле “Наименование препарата”;
* поле “Производитель”;
* поле “Основное действующее вещество”;
* поле “Количество действующего вещества”;
* поле “Цвет отметок в календаре”,
* кнопку “Вернуться назад”.
3. В поле “Цвет отметок в календаре” расположена кнопка  в виде стрелки вниз.  Нажатие кнопки открывает pop-up c  визуальным отображением возможных цветов.
4. После клика по выбранному цвету pop-up скрывается.
5. После нажатия на кнопку “Добавить”:
* препарат добавляется в перечень на странице “Мой Витамин”;
* открывается страница “Мой Витамин”.


 User story 4 Как зарегистрированный пользователь я хочу иметь возможность удалять препараты из списка пищевых добавок.
1. На странице “Мой витамин” в строке с наименованием препарата есть кнопка с изображением корзины.
2. После нажатия на кнопку “Корзина” на странице появляется pop-up.
3. Рop-up содержит: 
* текст “Вы действительно хотите удалить наименование препарата из списка” 
(наименование препарата - указано наименование согласно списку);
*  кнопку “Да”;
*  кнопку “Нет”;
        4. После нажатия кнопки “Да” pop-up cкрывается, препарат удаляется из списка.
5. После нажатия кнопки “Нет”  pop-up скрывается.


User story 5 Как зарегистрированный  пользователь я хочу иметь возможность отмечать в календаре дату приёма пищевой добавки. 
1. На странице “Мой витамин” в строке с наименованием препарата есть кнопка “Посмотреть календарь”.
2. После нажатия на кнопку “Посмотреть календарь” открывается страница “Календарь“.
3. Страница “Календарь” содержит :
* Заголовок календарь;
* текст “Наименование препарата”
* текст “Клик по дате добавит отметку о приеме препарата.
Повторный клик уберет отметку о приеме препарата.”
* кнопку “Запомнить”;
* секцию “Календарь” с отображением текущего месяца,
* кнопку “Вернуться назад”.
4. После клика мышки на выбранной дате появляется отметка (шестиугольник с заливкой соответствующего цвета , дата отображается).
5. После повторного клика мышки на дате с отметкой, отметка убирается.
6. После нажатия на кнопку “Запомнить”  информация сохраняется.
7. После нажатия на кнопку “Вернуться назад” открывается страница “Мой витамин“.


User story 6 Как зарегистрированный  пользователь я хочу видеть в календаре даты приема пищевых добавок, чтобы отслеживать период приёма препарата. 
1. На странице “Мой витамин” в строке с наименованием препарата есть кнопка “Посмотреть календарь”.
2. После нажатия на кнопку “Посмотреть календарь” открывается страница “Календарь“.
3. Календарь отображает текущий месяц.
4. В календаре выделена текущая дата (круг с заливкой синего цвета). 
5. Секция календарь имеет кнопки  “Меньше”, “Больше”.
6. При нажатии на кнопки  “Меньше”, “Больше” меняется месяц отображающийся в секции.
7. Даты секции “Календарь” имеют отметки, если они были заранее проставлены и сохранены. 




Критерии завершенности:


1. для кнопок:
1.1 цвет кнопки при наведении курсора мыши становится на один тон темнее


2. для полей:
2.1 поля принимают на ввод кириллицу, латинские буквы, заглавные и 
прописные буквы,  арабские цифры, точку, запятую, пробел, нижнее подчеркивание 
2.2  количество символов в поле:  минимальное - 2, максимальное - 50.
